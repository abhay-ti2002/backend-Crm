## Design Context

### Users
Two primary audiences with overlapping needs:
- **Support agents** (frontline): handling active tickets under time pressure — need fast scanning,
  clear next actions, and a system that feels like it has their back. Low cognitive friction is critical.
- **Managers/admins**: overseeing team performance and ticket flow — need data density,
  quick status reads, and hierarchy visibility.

### Brand Personality
**Professional · Reliable · Efficient** — every element earns its place. Confidence without arrogance.

### Aesthetic Direction
- Reference: Vercel / Clerk dashboards. Anti-reference: Jira-era density, consumer-app playfulness.
- Light-first + full dark mode. Indigo primary. Slate-900 sidebar. Functional status colors (red/amber/green).
- Space Grotesk headings · Jost body · Geist Mono for IDs/data.
- 150ms micro-interactions, 300ms layout transitions, ease-out-expo easing.

### Design Principles
1. Clarity over cleverness — established patterns only (badges, priority colors, steppers).
2. Density with breathing room — generous card padding, consistent gap-3/gap-4 spacing.
3. Trust through feedback — immediate visual state updates on every action.
4. Context always in reach — customer info, product, history inline, no extra navigation.
5. Status at a glance — color/badge communicates state before a word is read.
