## Design Context

### Users
Two primary audiences with overlapping needs:
- **Support agents** (frontline): handling active tickets under time pressure — need fast scanning,
  clear next actions, and a system that feels like it has their back. Low cognitive friction is critical.
- **Managers/admins**: overseeing team performance and ticket flow — need data density,
  quick status reads, and hierarchy visibility.

Both groups need **trust in the interface** — it should never feel ambiguous or overwhelming.

### Brand Personality
**Professional · Reliable · Efficient**
The CRM should feel like a tool built for serious work. No whimsy, no noise.
Every element earns its place. Confidence without arrogance.

### Aesthetic Direction
- **Reference**: Vercel / Clerk dashboards — clean whites, subtle shadows, confident
  type hierarchy, modern SaaS feel. Restrained but not sterile.
- **Anti-reference**: Avoid Jira-era density (visual clutter, too many borders, flat grays).
  Avoid consumer-app playfulness (rounded blobs, pastel overload, bouncy animations).
- **Theme**: Light-first with full dark mode. Indigo primary (`oklch(0.488 0.243 264.376)`).
  Slate-900 sidebar. Functional status colors (red/amber/green).
- **Typography**: Space Grotesk for headings (sharp, geometric authority),
  Jost for body (clean, legible at small sizes), Geist Mono for IDs and data.
- **Motion**: Purposeful and fast — 150ms micro-interactions, 300ms layout transitions.
  Smooth deceleration (`ease-out-expo`). Respects `prefers-reduced-motion`.

### Design Principles
1. **Clarity over cleverness** — if an agent has to think about what a UI element means, it failed.
   Use established patterns: status badges, priority colors, timeline steppers.
2. **Density with breathing room** — pack in the data, but use generous card padding and
   consistent spacing (gap-3/gap-4) so nothing feels cramped.
3. **Trust through feedback** — every action (resolve, escalate, accept) should feel confirmed.
   Immediate visual state updates, no silent failures.
4. **Context always in reach** — customer info, product details, ticket history should never
   require navigation. Surface them inline where the agent is already working.
5. **Status at a glance** — color-coded badges, availability indicators, and priority markers
   should communicate state before the agent reads a single word.
